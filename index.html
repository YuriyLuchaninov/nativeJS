WHAT IS A NATIVE APP?

developed for a specific OS...

  • REQUIRES INSTALLATION

  • NATIVE LOOK & FEEL (UX)

  • HAS ACCESS TO HARDWARE

  • PERFORMANCE / RESPONSIVENESS

... BUT!

Native code is one that's compiled to machine instructions and executed directly by the processor...

Any Java code for Android is not native by definition, because it's executed in a virtual machine.

Native development for Android is possible only with NDK.

with the exception of ART

LET'S START

PhoneGap

IDEA: THERE IS A BROWSER ON EVERY PLATFORM

PhoneGap SPECIFICS:

  • PhoneGap supports 13 platforms
  • It's compatible with NW, Crosswalk, CocoonJS, Ludei ...
  • It's just an SPA (on any framework)
  • Mobile browser (memory|GC|UI|UX)
  • UX/UI work the way you implement it
  • The biggest community
  • Native features demand native implementation
  • Almost immune to OS updates

IT IS JUST A BROWSER...

...WITH AN OPPORTUNITY OF CALLING NATIVE FUNCTIONALITY

Every OS has its browser

But! Android has more than 30 of them...

Titanium Mobile

IDEA: ONE CODE FOR ALL PLATFORMS

TITANIUM SPECIFICS:

  • MVC framework
  • Not all native functionality is supported
  • Compiled native UI
  • Work with native code via proxy classes
  • The platform is updated by the manufacturer
  • Architecture/approach changes regularly
  • A small community

I'M OLD, I'M SO OLD

React Native

IDEA: LEARN ONCE, WRITE FOR EVERYTHING

REACT NATIVE SPECIFICS:

  • FlexBox
  • ES6
  • Serialization of messages, everything is asynchronous
  • React and accompanying technologies
  • npm
  • The platform is updated by the manufacturer
  • 3rd-party native libraries / custom UI

TRENDING...

Welcome to the club © Titanium Appcelerator

Native Script

IDEA: DIRECT ACCESS TO NATIVE CODE

NATIVESCRIPT SPECIFICS:

  • Direct work with ANY native code
  • Work in UI flow
  • Common abstractions between platforms
  • Reflection, the platform doesn't need updates
  • MVVM
  • npm
  • Frequent releases

WHAT DO I DO WITH IT?

PERFECT CASES...

PhoneGap

  • Non-standard UI/UX (presentations, portfolios, digital kiosks, text)
  • Existing SPA
  • The fastest way of publishing in application stores
  • You need to cover more than Android and iOS

Titanium Mobile

  • You had experience with Titanium earlier
  • Implementation of some Titanium-specific native functionality
  • BaaS
  • Probably no other cases ...

ReactNative

  • You are acquainted with React and accompanying technologies
  • Standard, non-custom UI
  • You need it VERY fast development(JSX + CSS + components)
  • When was the last OS update? Long ago?

NativeScript

  • You need 3rd-party native libraries
  • You are ready to spend some more time on working with the native via JS
  • You understand there will be no full code sharing
  • You are not annoyed by MVVM or you are ready to learn Angular 2
